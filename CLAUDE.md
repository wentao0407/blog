# Claude 项目开发准则（自动全局加载）
适用于 JDK17 + SpringBoot3 + Vue3 + MySQL + Redis 项目
准则优先级：严谨 > 简洁 > 速度

================================================================================
# 一、编码前思考原则
1. 明确陈述你的假设前提。若有不确定之处，主动提问。
2. 若存在多种解读方式，一一列出，切勿擅自选择且不告知。
3. 若有更简洁的解决方案，务必说明。在合理情况下，敢于提出不同意见。
4. 若有内容理解不清，立即暂停。明确指出困惑点，然后提问。

================================================================================
# 二、代码简洁原则
1. 用最少的代码解决问题。不做任何投机性编写。
2. 不添加超出需求范围的功能。
3. 单一用途的代码不做抽象处理。
4. 不加入未被要求的 “灵活性” 或 “可配置性”。
5. 不可能出现的场景，不做错误处理。
6. 若你写了 200 行代码，但本可以用 50 行实现，就重写。
   自问：“资深工程师会认为这段代码过于复杂吗？” 如果答案是肯定的，就简化它。

================================================================================
# 三、精准修改原则
只改动必须改的部分。只清理自己造成的问题。
编辑现有代码时：
1. 不 “优化” 无关的周边代码、注释或格式。
2. 未出问题的代码，不重构。
3. 遵循现有代码风格，即便你有不同的写法。
4. 若发现无关的无效代码，只需指出 —— 不要删除。
5. 当你的修改导致某些代码成为无用项时：
6. 仅移除因你的修改而变得未被使用的导入项 / 变量 / 函数。
7. 除非被要求，否则不删除原本就存在的无效代码。
   检验标准：每一处修改的代码行，都必须能直接对应到用户的需求。

================================================================================
# 四、目标导向执行原则
明确成功标准。循环验证直至达成目标。
将任务转化为可验证的目标：
1. “添加校验” → “编写针对无效输入的测试用例，然后确保测试通过”
2. “修复漏洞” → “编写能复现漏洞的测试用例，然后确保测试通过”
3. “重构 X” → “确保重构前后测试均通过”

# 七、后端开发强制规范
1. 包结构固定：controller / service / mapper / entity / vo / dto / config / exception
2. 所有接口统一使用POST，除文件上传外，不准使用GET
3. 入参必须使用 VO/DTO，禁止直接使用实体类接收
4. 非空校验必须做，前后端一致
5. 表名必须以 t_ 开头
6. 复杂SQL（联表≥2张、子查询、聚合函数）必须加注释
7. 异常统一全局捕获，禁止 try-catch 吞异常
8. 日志必须使用 @Slf4j，不允许 System.out
9. 所有类、核心方法必须写JavaDoc注释
10. 关键业务逻辑必须加行内注释（说明为什么，不只是做什么）

================================================================================
# 八、前端开发强制规范
1. Vue组件使用PascalCase命名
2. 样式必须加 scoped
3. 禁止随意使用 any，必须标注类型
4. 接口统一使用 src/utils/request.js 发送
5. 按钮权限统一使用 v-hasPerm
6. 页面路由统一使用路由守卫
7. 表单必须做校验
8. 列表表头必须完整展示，不换行、不遮挡
9. 弹窗打开关闭正常，不残留状态
10. 所有按钮、交互必须可点击、可触发

================================================================================
# 九、测试与验收强制规范
1. 每个接口必须自测：正常、异常、边界值
2. 每个页面必须测试：所有按钮、所有输入、所有交互
3. 列表必须测试：查询、重置、分页、操作栏
4. 表单必须测试：提交、校验、清空、取消
5. 界面必须在 1920*1080 下正常显示
6. 测试完成必须给出结论：通过 / 不通过 + 原因
7. 不允许出现未测试就交付的功能

================================================================================
# 十、文档规范
1. 接口文档自动生成到：docs/api/
2. 文档必须与代码保持同步
3. 每个模块完成必须输出：功能说明 + 接口清单 + 使用说明
4. 不允许代码更新但文档不更新
